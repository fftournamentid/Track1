# PDF Android fix — copy to stable URI before opening

## The Rule
After `Print.printToFileAsync`, copy the returned temp URI to `FileSystem.documentDirectory` before calling `FileSystem.getContentUriAsync`. Never pass the raw temp URI directly to `getContentUriAsync`.

## Why
`getContentUriAsync` requires the file to be in a directory backed by a FileProvider content authority. The temp directory used by `expo-print` is NOT in that authority on most Android devices, causing "Cannot open file" errors.

## How to Apply
```ts
const dest = `${FileSystem.documentDirectory}invoice_preview.pdf`;
await FileSystem.copyAsync({ from: result.uri, to: dest });
const contentUri = await FileSystem.getContentUriAsync(dest);
await IntentLauncher.startActivityAsync('android.intent.action.VIEW', {
  data: contentUri, flags: 1, type: 'application/pdf',
});
```

Always wrap in try/catch and fall back to `Sharing.shareAsync(stableUri, { mimeType: 'application/pdf' })` if IntentLauncher throws (device has no PDF viewer installed).

## Where it's applied
`artifacts/mobile/services/pdfService.ts` — `openPDF()` and `openInvoicePDF()` functions.
